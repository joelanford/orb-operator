# Operator Infrastructure

## Summary

Stand up the operational infrastructure so the operator can be built, deployed to a local kind cluster, and run with real (but empty) CRDs. This bridges the gap between the project scaffolding (which produces a compiling skeleton) and the first controller implementation (which needs CRDs registered and a working deploy pipeline).

After this work item, `make run` produces a running operator pod in a kind cluster with three registered CRDs, an HTTPS metrics endpoint secured by authn/authz, and cluster-admin RBAC.

## Design

### Empty API type stubs

Three cluster-scoped CRD types in `api/v1alpha1/`:

| Type | Resource | Short |
|---|---|---|
| `ClusterObjectSet` | `clusterobjectsets` | `cos` |
| `ClusterObjectSetRevision` | `clusterobjectsetrevisions` | `cosr` |
| `ClusterObjectSlice` | `clusterobjectslices` | `cosl` |

Each type has empty `Spec` and `Status` structs. The full ADR-0001 fields are added later in the "API Types" work item. Controller-gen markers produce CRDs and deepcopy.

Scheme registration follows the standard apimachinery pattern: `runtime.NewSchemeBuilder` + `AddToScheme` in a `groupversion_info.go`, with a centralized `addKnownTypes` function registering all types.

A `go:generate` directive in `doc.go` invokes controller-gen for CRD output (`deploy/crds/`) and deepcopy generation.

### HTTPS metrics endpoint

The controller-runtime manager is configured with:
- `SecureServing: true` on the metrics server
- `FilterProvider: filters.WithAuthenticationAndAuthorization` for the authn/authz middleware on `/metrics`
- Bind address `:8443`

The manager also registers the v1alpha1 scheme so it can watch the CRD types once controllers are added.

### Jsonnet operator manifests

`deploy/operator.jsonnet` renders a complete deployment manifest (as a JSON List) parameterized by two external variables:

| Variable | Default | Purpose |
|---|---|---|
| `image` | *(required, no default)* | Container image reference |
| `namespace` | `orb-operator-system` | Target namespace |

Rendered resources:

1. **Namespace** — `orb-operator-system`
2. **ServiceAccount** — `orb-operator` in the operator namespace
3. **ClusterRoleBinding** — binds the SA to the built-in `cluster-admin` ClusterRole
4. **Deployment** — single replica, `orb-operator` container on port 8443, running as non-root (UID 65532)
5. **Service** — `orb-operator-metrics`, exposing port 8443 for metrics scraping

CRDs are imported from `deploy/crds/` (generated by controller-gen) and included in the jsonnet output.

### Makefile targets

New targets added to the existing Makefile:

| Target | What it does |
|---|---|
| `run` | Builds image, creates kind cluster (if needed), loads image, applies manifests |

The `IMAGE` variable defaults to `ghcr.io/joelanford/orb-operator:dev`. `make run` is the full local dev loop.
